Useprint is a React-first toolkit for building printable documents as actual React components.

The docs are a WIP and will be improved upon. This library is a very narrow fork from [React Email](https://react.email) and optimized for making documents. This library has mostly been used internally, so I apologize if the docs are a bit lackluster, but I have been using the library internally and it has improved our PDF DX tremendously.

The core flow is simple:

1. Build your document with `@useprint/components`
2. Preview it locally with `@useprint/cli`
3. Render it to HTML with `@useprint/render`
4. Turn that HTML into a PDF with `useprint-js`[^useprint-js] or your own Chromium setup

[^useprint-js]: `useprint-js` is not publicly available yet but is coming soon. Join the [Useprint Discord](https://discord.gg/MEKzQGub) for updates.

## Why Useprint

- Author documents in React instead of string templates.
- Keep layout logic close to the component tree that produces it.
- Preview documents locally while you iterate.
- Render portable HTML that works with your own PDF backend or `useprint-js`[^useprint-js].
- Because the PDF path is Chromium-based, you are not limited to a bespoke or partial CSS engine—standard web layout and styling apply.
